**JSONL** 是 **JSON Lines** 的缩写。它是一种**文本文件格式**，每行包含一个独立的、合法的 JSON 对象。

简单理解：**一个文件里，一行就是一个 JSON 数据**。

### 核心特征

1.  **每行一个 JSON**：文件中的每一行都是一个完整的、标准的 JSON 对象（用 `{}` 包裹）。
2.  **行分隔符是换行符**：使用 `\n`（换行符）来分隔每条数据记录。
3.  **文件整体不是合法 JSON**：整个文件内容 `{...}\n{...}\n...` 本身 **不是** 一个合法的 JSON 数组或对象。这使它区别于传统的单个 JSON 数组文件 `[{...},{...}]`。

### 示例

一个名为 `data.jsonl` 的文件内容可能长这样：
```jsonl
{"id": 1, "name": "Alice", "score": 95}
{"id": 2, "name": "Bob", "score": 87}
{"id": 3, "name": "Charlie", "score": 92}
```
每行都是一个独立的记录，行尾没有逗号。

### 为什么使用 JSONL？（优势）

-   **流式处理**：可以逐行读取文件，不必一次性加载整个文件到内存。非常适合处理**大型数据集**（例如几十GB的日志或训练数据）。
-   **易于追加**：想添加新数据时，只需在文件末尾新起一行写入新的 JSON 对象即可，非常方便。而标准的 JSON 数组格式要追加数据，通常需要解析、修改整个数组结构。
-   **容错性好**：如果文件某一行损坏（比如缺少括号），不影响读取其他行。而单个 JSON 文件一个语法错误就会导致整个文件无法解析。
-   **语言支持广泛**：任何能解析 JSON 和处理逐行文本的编程语言都能轻松处理。例如在 Python 中：
    ```python
    import json

    # 逐行读取 JSONL 文件
    with open('data.jsonl', 'r') as f:
        for line in f:
            data = json.loads(line)
            print(data["name"])
    ```

### 常见用途

-   **机器学习数据集**：如 Hugging Face 的 `datasets` 库常用 `.jsonl` 格式存储训练数据。
-   **日志文件**：每行一条日志记录（通常比较长，结构化为 JSON）。
-   **数据交换**：在两个系统间批量传输记录，尤其是一边产生一边消费的场景。

### JSONL vs. JSON 数组

| 特性 | JSONL (每行一个 JSON) | 标准 JSON 数组 |
| :--- | :--- | :--- |
| **文件整体** | 不是合法 JSON | 是合法 JSON `[ ... ]` |
| **读取方式** | 逐行流式读取 | 需完整解析整个数组 |
| **内存占用** | 极低（只读一行） | 高（需加载全部数据） |
| **追加数据** | 直接 `echo {...} >> file.jsonl` | 困难（需重写整个数组） |
| **容错性** | 一行损坏不影响其他行 | 任何一处错误全盘皆输 |
| **典型场景** | 大数据、日志流、持续写入 | 配置文件、小型静态数据 |

总结：**如果你需要逐条处理海量 JSON 数据，或者数据是不断产生的流式数据，JSONL 是非常实用的格式。**
